# VAKA OS Master Programme Blueprint

**Edition:** 1.0  
**Status:** Architecture and programme baseline  
**Effective date:** 2026-07-11  
**Owner:** VAKA leadership and Architecture Office

This collection converts the supplied Book Two to Book Four chat material and the proposed 24-volume catalogue into a governed, repository-ready construction system. It defines the complete target architecture, delivery controls, capability boundaries, acceptance gates, and mission catalogue for VAKA OS.

It is a **complete blueprint**, not a claim that the complete product has already been implemented. Every capability carries separate definition, implementation, verification, and availability states. Only linked evidence may advance those states.

## How to use it

1. Start with [Document Control](00-governance/DOCUMENT-CONTROL.md) and the [status legend](00-governance/STATUS-AND-CLAIM-LEGEND.md).
2. Read Books 1-4 for product scope, platform construction, programme sequencing, and engineering execution.
3. Use Books 5-23 as the target capability and readiness specifications.
4. Execute only approved Mission Packs registered by Book 24.
5. Use [MANIFEST.md](MANIFEST.md) to generate the consolidated Markdown and PDF in the fixed order.

## Authority

This collection is governed by `AGENTS.md`, the VAKA Constitution, accepted ADRs, the authoritative finance architecture, and the precedence in `knowledge-system/18-decisions/ADR-001-document-authority.md`. Detailed source documents retain authority where linked. The consolidated PDF is a controlled presentation artifact, not an independently editable source.
